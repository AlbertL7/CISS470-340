Bridging the Gap: From Controls to Configuration

NIST SP 800-70 is the crucial implementation bridge that transforms abstract security controls into concrete technical configurations. While FIPS 199, FIPS 200, and NIST 800-53 tell you what security controls you need, NIST 800-70 shows you how to implement them on actual IT products.

The National Checklist Program (NCP) provides standardized, tested configuration checklists that help organizations implement security controls consistently and effectively across their technology infrastructure. These checklists save organizations significant time and resources while reducing configuration errors and security gaps.

Key Concepts

Configuration Baseline Standardized secure settings for IT products
Checklist Repository Centralized database of validated checklists
SCAP Content Machine-readable security configurations
Tier System Four-tier checklist complexity model

üìö National Checklist Program (NCP) Structure

The NCP uses a tiered approach to categorize checklists based on their automation level and complexity. This helps organizations select appropriate checklists based on their technical capabilities and requirements.

Tier I - Prose

Human-readable guidance in narrative form

  • PDF or Word documents
  • Manual implementation required
  • Good for small organizations
  • Example: CIS Benchmarks PDFs

Tier II - Generalized

Configuration files with manual review needed

  • XML, INI, or config files
  • Some automation possible
  • Requires customization
  • Example: Group Policy templates

Tier III - Technology Specific

Automated scripts for specific products

  • PowerShell, Bash scripts
  • Direct implementation
  • Product-specific automation
  • Example: DISA STIG scripts

Tier IV - SCAP Enabled

Fully automated scanning and remediation

  • XCCDF, OVAL content
  • Automated compliance checking
  • Continuous monitoring capable
  • Example: SCAP data streams

Fix issues based on CAT severity priorities

CAT I (Critical)

Fix immediately

Examples: Default passwords, critical patches missing

CAT II (High)

Fix within 30 days

Examples: Weak encryption, insufficient logging

CAT III (Medium)

Fix within 90 days

Examples: Banner messages, documentation

-- Remove unnecessary default accounts
DROP USER SCOTT CASCADE;
DROP USER HR CASCADE;
DROP USER OE CASCADE;

-- Implement least privilege
REVOKE CREATE SESSION FROM PUBLIC;
REVOKE CREATE TABLE FROM CONNECT;

-- Enable unified auditing
ALTER SYSTEM SET unified_audit_trail=DB,EXTENDED SCOPE=SPFILE;

üîß SCAP: Automation Protocol for NIST 800-70

The Security Content Automation Protocol (SCAP) enables automated vulnerability management, security measurement, and policy compliance evaluation. It's the technical foundation for Tier IV checklists in NIST 800-70.

XCCDF

eXtensible Configuration Checklist Description Format - Describes security checklists

OVAL

Open Vulnerability Assessment Language - Defines system states and configurations

CPE

Common Platform Enumeration - Identifies IT products and platforms

CVE

Common Vulnerabilities and Exposures - Standard vulnerability identifiers

CVSS

Common Vulnerability Scoring System - Rates vulnerability severity

CCE

Common Configuration Enumeration - Identifies security configurations

5. How does NIST 800-70 relate to the other standards?

It replaces FIPS 199, FIPS 200, and NIST 800-53
It's an optional supplement for advanced users
It provides implementation guidance for controls selected through the other standards
